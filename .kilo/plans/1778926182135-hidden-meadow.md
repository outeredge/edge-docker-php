# Plan: Move FrankenPHP Supervisor Variant to Base Image

## Objective
Create a new base image variant (`-frankenphp-super`) that includes `supervisor` (and related services like `valkey` and `cloud-sql-proxy`), allowing production workloads to optionally run multiple processes, while keeping the standard `-frankenphp` image lightweight and single-process. The development image will then inherit from this new variant.

## Changes in Base Image (`/workspaces/edge-docker-php`)
1. **Create Dockerfile Variant Files**:
   - `Dockerfile.php83-frankenphp-super`
   - `Dockerfile.php84-frankenphp-super`
   *(These will build `FROM outeredge/edge-docker-php:8.X-frankenphp` and `INCLUDE+ Dockerfile.common-frankenphp-super`)*
2. **Create `Dockerfile.common-frankenphp-super`**:
   - Install `supervisor`, `valkey`, `cron` (optional).
   - Install `cloud-sql-proxy` from the Google container.
3. **Add Supervisor Config & Launch Script**:
   - Create a static `templates/supervisord-frankenphp.conf` defining `frankenphp`, `valkey`, and `sql-proxy` (running unconditionally without jinja).
   - Create `launch-frankenphp-super.sh` which simply starts `supervisord`.
4. **Update `publish.sh`**:
   - Add the docker build/push commands for the new `-frankenphp-super` variants.

## Changes in Development Image (`/workspaces/edge-docker-php-dev`)
1. **Update Base Images**:
   - Change `Dockerfile.php83-frankenphp` and `Dockerfile.php84-frankenphp` to use `FROM outeredge/edge-docker-php:8.X-frankenphp-super`.
2. **Clean Up `Dockerfile.common-frankenphp`**:
   - Remove `supervisor`, `valkey` from the `apt-get install` list.
   - Remove the `cloud-sql-proxy` installation.
3. **Clean Up Configs**:
   - Remove `etc/supervisord.conf` (since the base image will provide it via templates).
4. **Update `dev.sh`**:
   - Modify `dev.sh` to chain into the new base image launch script (`/launch-frankenphp-super.sh`) instead of starting `supervisord` manually.
